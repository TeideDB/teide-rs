SQL REFERENCE

Table Functions

Load external data sources directly in SQL queries.

read_csv()

Load a CSV file as a table:

-- Direct query from CSV
SELECT * FROM read_csv('data.csv');

-- Create a persistent table
CREATE TABLE t AS SELECT * FROM read_csv('data.csv');

Note: CSV files are automatically type-inferred. Teide samples the file to determine column types.

The Rust API provides additional options:

Context::read_csv_opts(path, delimiter, header, col_types)
  • delimiter: Column separator character (default: ,)
  • header: Whether the first row is a header (default: true)
  • col_types: Optional explicit type specification per column

read_splayed()

Open a splayed (column-per-file) table with zero-copy memory mapping:

SELECT * FROM read_splayed('/path/to/table');

A splayed table directory contains:

  • A .d metadata file
  • One file per column (binary columnar format)
  • An optional shared symbol file

read_parted()

Open a date-partitioned table:

SELECT * FROM read_parted('/path/to/db', 'table_name');

Partitioned tables store data in date-based directories:

db_root/
  sym              -- shared symbol table
  2024-01-15/
    trades.d       -- metadata
    price          -- column file
    qty            -- column file
  2024-01-16/
    trades.d
    price
    qty

Note: Results from read_parted() are cached internally to avoid re-reading on subsequent calls.
